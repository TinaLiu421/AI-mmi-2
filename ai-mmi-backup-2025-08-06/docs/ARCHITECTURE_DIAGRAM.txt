================================================================================
                         API ENDPOINT SUMMARY
================================================================================

POST /api/documents/upload
├─ Input: FormData with file
├─ Processing: Extract text → Analyze with Gemini
└─ Output: { success, document, analysis }

GET /api/documents/{id}
├─ Input: Document ID
├─ Processing: Retrieve from DB
└─ Output: Document details + analysis

GET /api/documents
├─ Input: member_id, status, limit, offset
├─ Processing: Query with filters and pagination
└─ Output: Array of documents + pagination info

DELETE /api/documents/{id}
├─ Input: Document ID
├─ Processing: Delete file + DB record
└─ Output: { success, message }

POST /api/documents/{id}/reanalyze
├─ Input: Document ID + analysis_type
├─ Processing: Re-analyze extracted text
└─ Output: { success, analysis }


================================================================================
                           ERROR HANDLING FLOW
================================================================================

Invalid File
    ├─ Size > 10MB → Return 422 "File too large"
    ├─ Wrong MIME → Return 422 "Invalid file type"
    └─ Bad extension → Return 422 "Invalid extension"

Extraction Failed
    ├─ Can't extract text → Mark status "failed"
    ├─ Return error message → "Could not extract text"
    └─ Store error_message in DB

Analysis Failed
    ├─ Gemini API error → Mark status "failed"
    ├─ Store error message → "API timeout"
    └─ Return error response

Network Error
    ├─ Connection fails → Alert user
    ├─ Auto-clear file input
    └─ Enable retry

Database Error
    ├─ Log to storage/logs/laravel.log
    ├─ Return 500 error
    └─ Alert user
